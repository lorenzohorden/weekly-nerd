Rule of Least Power

Kilian heeft het in zijn lezing vooral gehad over de "rule of least power", dat houd in dat de beste oplossing voor een probleem de minst complexe oplossing is. Een hoop problemen waar mensen JavaScript voor gebruiken zijn nu best makkelijk op te lossen met CSS, omdat CSS minder krachtig is dan JavaScript is het dus beter om hier CSS voor te gebruiken.

Hij gaf hier een hoop voorbeelden van en ik heb hier een klein overzicht:

  • Html {scroll-behavior: smooth;} voor langzaam scrollen
  • Input type color als color picker
  • dialog element is een vervanger van alert, prompt en confirm
  • Input, textarea {field-sizing: content} om input en textarea grootte aan te passen aan de hand van de inhoud (werkt alleen nog in chrome)

Mijn mening over de lezing:

Ik vind the rule of least power een super interessante kijk naar web development en het heeft me aan het denken gezet over hoe vaak ik een JavaScript code heb geschreven om een ingebouwde HTML functionaliteit na te maken, dat heeft me achteraf veel tijd gekost die ik er niet in had hoeven steken. Ik heb na deze lezing een youtube video gekeken over alle html elementen die mensen over het hoofd zien om te kijken of er dingen zijn die ik vaker kan gebruiken en heb veel geleerd over HTML.

Deze lezing was erg interessant en heeft mijn kijk naar programmeren veranderd, normaal gebruikte ik heel vaak JavaScript en CSS om een paar HTML elementen te laten werken op de manier die ik wil, maar HTML bied heel veel standaard functionaliteit die ik weinig gebruik wat ik in de toekomst zeker anders wil doen.
